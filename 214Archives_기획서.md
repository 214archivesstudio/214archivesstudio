## 1. 프로젝트 개요 (Project Overview)

### 1.1 프로젝트 목표

사진 작가 및 촬영 감독의 작품(사진/영상)을 최상의 퀄리티와 속도로 보여주는 포트폴리오 웹사이트를 구축합니다. 시각적 임팩트와 우수한 사용자 경험을 결합하여 작가의 창작 세계를 효과적으로 전달하는 것을 목표로 합니다.

| 구분 | 내용 |
|------|------|
| **프로젝트명** | 214Archives Studio 포트폴리오 웹사이트 |
| **클라이언트** | 214Archives Studio (사진작가 / 촬영감독) |
| **사이트 유형** | 포트폴리오 웹사이트 (사진 + 영상) |
| **주요 타겟** | 작품 의뢰 클라이언트, 업계 관계자, 일반 방문자 |
| **디자인 컨셉** | 다크 톤 기반 미니멀 디자인, 시네마틱 영상미학적 분위기 |
| **레퍼런스 사이트** | [ethanandtom.com](https://www.ethanandtom.com/), [cubiflow.com](https://www.cubiflow.com/) |
| **핵심 요소** | 빠른 로딩, 유지비용 최소화, 모바일 최적화, 인터랙티브 애니메이션 |

### 1.2 핵심 가치

- **시각적 몰입감:** 작품이 주인공이 되는 풀스크린 미디어 표현으로 방문자의 몰입도를 극대화합니다.
- **성능 우선:** 고해상도 이미지와 영상을 빠르게 전달하기 위한 최적화된 로딩 전략을 적용합니다.
- **직관적 UX:** 복잡한 탐색 없이 작품을 자연스럽게 탐색할 수 있는 직관적인 사용자 경험을 제공합니다.
- **반응형 디자인:** PC, 태블릿, 모바일 모든 환경에서 최적의 경험을 보장합니다.
- **운영 효율성:** 유지비용을 최소화하고 콘텐츠 관리가 용이한 구조를 구축합니다.

## 2. 사이트맵 (Sitemap)

### 2.1 전체 페이지 구조

| 페이지 (URL) | 설명 | 주요 기능 |
|-------------|------|----------|
| **Home** (`/`) | 로딩 애니메이션 + 메인 페이지 | 로고 트랜지션, 배경 영상, 네비게이션 |
| **Showreel** (`/showreel`) | 영상 작업물 갤러리 | 썸네일 그리드 + 배경 영상 재생 |

### 2.2 공통 컴포넌트

- **헤더 (Header):** 좌측 상단에 스튜디오명과 태그라인, 우측 상단에 214 로고 배치. 투명 배경으로 콘텐츠 위에 배치됩니다.
- **네비게이션 (Navigation):** PC에서는 상단 가로 메뉴로 표시, 모바일에서는 햄버거 메뉴로 전환되며 클릭 시 세로 메뉴가 펼쳐집니다.
- **배경 레이어 (Background):** 모든 페이지의 배경으로 뉴욕 스카이라인 이미지/영상이 깔리며, 호버 시 동적으로 변경됩니다.
- **로딩 애니메이션 (Loading):** 최초 접속 시 214 로고가 하단에서 상단으로 올라오며 페이드인되는 인트로 트랜지션이 적용됩니다.

## 3. 페이지별 상세 기획 (Page Specifications)

### 3.1 홈 / 로딩 화면 (Home / Loading)

#### 3.1.1 로딩 애니메이션

홈페이지 최초 접속 시 로딩 애니메이션이 재생되며, 이후 메인 페이지로 전환됩니다.

1. 검은 배경 화면에서 시작합니다.
2. "214" 로고가 화면 하단에서 상단으로 올라오며 페이드인됩니다.
3. 로고가 완전히 나타난 후 배경이 메인 페이지 콘텐츠로 전환됩니다.
4. 배경 영상/이미지가 페이드인되며 네비게이션 메뉴가 표시됩니다.

> **참고:** ethanandtom.com의 로딩 트랜지션 및 cubiflow.com의 로고 애니메이션 스타일을 참고합니다.

#### 3.1.2 메인 페이지 레이아웃

메인 페이지는 풀스크린 배경 영상/이미지 위에 네비게이션이 오버레이되는 구조입니다.

- **좌측 상단:** "214Archives Studio" 텍스트 + "Every moment happens once. We archive it!" 태그라인
- **우측 상단:** 214 로고 이미지
- **중앙 상단:** `Showreel` | `Archives` | `Film` | `Photography` | `Personal Work` | `Contact` 메뉴
- **배경:** 뉴욕 스카이라인 풀스크린 이미지 (어두운 오버레이 적용)

#### 3.1.3 모바일 메뉴

모바일 환경에서는 햄버거 버튼을 통해 세로 메뉴가 표시됩니다. 메뉴 클릭 시 배경이 어두워지며 중앙에 세로로 메뉴 항목이 나열됩니다. 메뉴 항목에는 Showreel, Archives, Film, Photography, Personal Work, Contact가 포함됩니다.

---

### 3.2 Showreel 페이지

#### 3.2.1 개요

영상 작업물을 보여주는 페이지로, 썸네일 기반 그리드 형식으로 영상을 탐색할 수 있습니다.

#### 3.2.2 기능 상세

| 기능 | 상세 설명 | 참고 사항 |
|------|----------|----------|
| **썸네일 그리드** | 영상 작업물의 대표 썸네일을 그리드 형식으로 배치. 각 썸네일에는 작업물 제목이 표시됨. | 예: "2025 Showreel" |
| **배경 영상 재생** | 썸네일에 마우스 호버 시 페이지 전체 배경에 해당 영상이 무음으로 재생됨. 호버 해제 시 기본 배경으로 복귀. | ethanandtom.com 참고 |
| **상세 페이지** | 썸네일 클릭 시 상세 페이지로 이동. YouTube/Vimeo 임베드 플레이어를 통해 영상 재생 가능. | 영상은 외부 플랫폼 활용 |

#### 3.2.3 인터랙션 플로우

1. 사용자가 Showreel 페이지에 진입합니다.
2. 썸네일 그리드가 표시됩니다 (배경: 기본 이미지).
3. 썸네일에 마우스를 올리면 배경 전체가 해당 영상으로 전환되며 무음 재생됩니다.
4. 썸네일을 클릭하면 상세 페이지로 이동하여 영상을 재생할 수 있습니다.

## 4. 디자인 시스템 (Design System)

### 4.1 컬러 팔레트

| 용도 | HEX | 설명 |
|------|-----|------|
| **배경** | `#1A1A1A` | 다크 배경 |
| **텍스트** | `#FFFFFF` | 흰색 텍스트 |
| **오버레이** | `rgba(0,0,0,0.4~0.6)` | 사진 위 오버레이 |
| **액센트** | `#CCCCCC` | 보조 텍스트/구분선 |

### 4.2 타이포그래피

- **제목 (Heading):** 산세리프 계열 폰트 (Pretendard, Noto Sans KR 등) / Bold / 흰색
- **본문 (Body):** 산세리프 계열 / Regular / 흰색 또는 밝은 회색
- **태그라인:** 이탤릭 스타일 / Light / 밝은 회색
- **메뉴:** 산세리프 계열 / Medium / 흰색, 호버 시 미묘한 크기 변화 또는 언더라인 효과

### 4.3 애니메이션 및 인터랙션

- **페이지 전환:** 부드러운 페이드 전환 효과 (Framer Motion 활용)
- **호버 효과:** 썸네일 호버 시 `scale(1.03)` + 오버레이 변화 + 배경 전환
- **스크롤 애니메이션:** 콘텐츠 진입 시 페이드인 + 슬라이드업 효과
- **로딩 애니메이션:** 로고 상승 + 페이드인 조합
- **모바일 제스처:** 좌우 스와이프로 갤러리 탐색 지원

## 5. 기술 아키텍처 (Technical Architecture)

### 5.1 기술 스택

| 구분 | 기술/서비스 | 선정 사유 |
|------|-----------|----------|

| **언어** | TypeScript | 타입 안정성, 유지보수 용이 |
| **UI 라이브러리** | React 18+ | 컴포넌트 기반 UI 구축 |

| **애니메이션** | Framer Motion | 페이지 전환, 호버 효과, 스크롤 애니메이션 |
| **영상 호스팅** | YouTube / Vimeo | 외부 영상 스트리밍으로 서버 비용 절감 |
| **이미지 최적화** | Next.js Image + Cloudinary | 자동 WebP 변환, 레이지 로딩, CDN 배포 |
| **배포** | Vercel | 글로벌 CDN, 자동 배포, 무료 호스팅 |

| **CMS (옵션)** | Sanity / Notion API | 콘텐츠 관리 시스템 연동 가능 |

### 5.2 호스팅 및 비용 전략

| 구분 | 서비스 | 비용 | 비고 |
|------|-------|------|------|
| **웹사이트 배포** | Vercel (Global CDN) | **0원 (무료)** | 개인 포트폴리오 용도 무료 사용 가능 |
| **이미지 호스팅** | Cloudinary (Image CDN) | **0원 (초기)** | 월 25GB 대역폭까지 무료 |
| **영상 호스팅** | YouTube / Vimeo | **0원** | 영상 직접 호스팅 불필요 |
| **도메인** | 사용자 구매 | **연 1~2만원** | .com 도메인 기준 |

초기 세팅 후 월간 유지비용은 도메인 비용(연 1~2만원)을 제외하면 기본적으로 0원입니다. 트래픽이 크게 증가하는 경우에만 추가 비용이 발생할 수 있습니다.

### 5.3 미디어 최적화 전략

- **이미지:** 원본 사진을 WebP 포맷으로 자동 변환하여 용량 80~90% 절감 (화질 저하 최소화)
- **썸네일:** 저해상도 썸네일을 별도 생성하여 목록 페이지 로딩 속도 향상
- **레이지 로딩:** 스크롤 위치에 따라 이미지를 순차적으로 로드하여 초기 로딩 속도 개선
- **영상:** YouTube/Vimeo 임베드 활용으로 서버 비용 0원 유지
- **배경 영상:** 호버 시에만 로드되도록 지연 로딩 처리

## 6. 반응형 디자인 (Responsive Design)

### 6.1 브레이크포인트

| 디바이스 | 화면 크기 | 레이아웃 | 비고 |

- **Archives:** PC에서는 3열 그리드, 태블릿 2열, 모바일 1열 세로 스크롤
- **Photography:** PC에서는 가로 슬라이드, 모바일에서는 세로 스크롤로 전환
- **Archives 상세:** PC에서는 4열 x 3행 그리드, 모바일에서는 세로 스크롤로 전체 화면 채움
- **네비게이션:** PC 가로 메뉴 → 모바일 햄버거 + 세로 메뉴

## 9. 고려사항 및 전달사항

### 9.1 클라이언트 확인 필요 사항

1. **도메인:** 사용할 도메인이 있는지, 새로 구매해야 하는지 확인이 필요합니다.
2. **SNS 링크:** Contact 페이지에 연결할 정확한 SNS URL 목록이 필요합니다 (Instagram, Vimeo, 전화번호, 이메일 등).
3. **미디어 원본:** 업로드할 사진 및 영상의 원본 파일을 정리하여 제공해 주셔야 합니다.
4. **영상 플랫폼:** Showreel 영상이 업로드된 YouTube/Vimeo 링크가 필요합니다.
5. **로고 파일:** 214 로고의 고해상도 원본 파일(SVG 또는 PNG)이 필요합니다.
6. **Film 페이지:** 추후 콘텐츠 추가 일정 및 범위를 확정해 주셔야 합니다.

### 9.2 기술적 고려사항

- **배경 영상 성능:** 호버 시 배경에 영상을 재생하는 기능은 데이터 사용량과 성능에 영향을 줄 수 있으므로, 영상 품질과 용량의 균형을 맞춰야 합니다.
- **이미지 수량:** Archives 섹션에 많은 사진이 포함되므로 페이지네이션 또는 무한 스크롤 전략이 필요합니다.
- **브라우저 호환성:** 주요 브라우저(Chrome, Safari, Firefox, Edge) 및 iOS/Android 모바일 브라우저에서의 호환성 테스트가 필요합니다.
- **콘텐츠 확장성:** 향후 작업물이 추가될 때 쉽게 확장할 수 있는 데이터 구조와 컴포넌트 설계가 필요합니다.
